# claw — agent guide

`claw` is a Bun + `argc` CLI: an OKF-style markdown knowledge indexer/reader.
Four commands — `index`, `read`, `install`, `uninstall`.

## Conventions

- **Schema-first.** `src/schema.ts` is the public contract — command names,
  flags, examples, and help all live there. Change it deliberately.
- **Output is YAML by default**, `--json` for pipes. Errors are structured
  (`src/errors.ts`) with exit codes 2 (usage) / 3 (not found) / 4 (forbidden).
- **`bun run check`** (fmt:check + typecheck + test) must stay green. Use
  `oxfmt` and `tsgo`; never introduce eslint / prettier / tsc.

## Architecture

- Entry/wiring: `src/main.ts`. Command handlers: `src/handlers/*`. Everything
  else in `src/` is reusable logic.
- `wiki.ts` — `scanDocs` (enumerates via `git ls-files`, glob fallback),
  `buildIndex` (a YAML list), and the AGENTS.md inline-block injector.
- `markdown.ts` — markdown-it backed `--toc` / `--section` / summary. Output is
  validated byte-for-byte against the `ctx` tool; keep it aligned.
- `hooks.ts` — wires `claw index --inject` into Claude Code / Codex lifecycle
  hooks. The "freshness" mechanism: synchronous, stateless, no daemon.

## Invariants — don't break these silently

- **No on-disk index artifact.** `claw index` prints to stdout. `--inject`
  embeds the index inline into a host file (e.g. `AGENTS.md`); that block IS
  the index. There is no `index.yaml` file.
- `scanDocs` indexes only frontmatter-bearing markdown. Dot-prefixed
  directories (`.git`, `.claw`, `.scratch`, `.agents`, `.claude`, …) are
  Unix-hidden infrastructure and are never indexed.
- `claw index --inject` is idempotent: a no-op rebuild must not write the host
  file. Hooks fire frequently; mtime churn would wake every other watcher.
- Tests are the contract. Changing behavior without a test usually means it
  isn't locked.
